# reCAPTCHA v3 Integration Gotchas (Cheatsheet)

## 🚨 CRITICAL: The Script Tag
Always ensure the user imports `enterprise.js`, not `api.js`, when using Enterprise keys.
Using `api.js` with an Enterprise key results in a `MALFORMED` invalid reason when `createAssessment` is called.

```html
<!-- ✅ RIGHT: reCAPTCHA Enterprise -->
<script src="https://www.google.com/recaptcha/enterprise.js?render=YOUR_SITE_KEY"></script>

<!-- ❌ WRONG: Standard (Throws MALFORMED) -->
<script src="https://www.google.com/recaptcha/api.js?render=YOUR_SITE_KEY"></script>
```

## Duplicate Form Field (Array Token Error)
When injecting a hidden input to hold the token, always check if one already exists.
Otherwise, on form resubmission, multiple inputs are created. On the backend, this turns into `token: ["token1", "token2"]` instead of a string, causing validation to fail.

```javascript
// Example defense to prevent duplicate DOM fields
let input = form.querySelector('input[name="g-recaptcha-response"]');
if (!input) {
  input = document.createElement("input");
  input.type = "hidden";
  input.name = "g-recaptcha-response";
  form.appendChild(input);
}
input.value = token;
```

## Action Names
`grecaptcha.enterprise.execute` takes an `action` config.
Important verification rules for action names:
1. They must match what backend checks for (e.g. `expectedAction: 'checkout'`).
2. They may only contain alphanumeric chars, slashes `/,` and underscores `_`.
3. They must NOT be user-specific (do not include the username or session ID in the action name).
